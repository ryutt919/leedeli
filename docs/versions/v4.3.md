### 1-2. Supabase DB 마이그레이션 ✅

두 개의 마이그레이션을 Supabase MCP로 직접 적용:

#### migration 004: admin_users RLS 무한 재귀 수정

**문제**: `admin_users` SELECT 정책이 자기 자신을 EXISTS 서브쿼리로 참조 → PostgreSQL `42P17 infinite recursion detected`

```sql
-- 기존 (버그): EXISTS (SELECT 1 FROM admin_users WHERE ...)
-- 수정: 직접 비교
CREATE POLICY "admin_users_select_own" ON public.admin_users
  FOR SELECT USING (user_id = auth.uid());
```

#### migration 005: preps.data 컬럼 추가

**문제**: `preps` 테이블이 `id, name, updated_at` 구조였으나 `prepsRepo.ts`는 `data jsonb` 컬럼 기대 → `42703 column does not exist`

```sql
ALTER TABLE public.preps
  ADD COLUMN IF NOT EXISTS data JSONB NOT NULL DEFAULT '{}'::jsonb,
  ADD COLUMN IF NOT EXISTS created_at TIMESTAMPTZ NOT NULL DEFAULT now();
```

**prepsRepo.ts 추가 수정**: upsert 시 `name` 필드 포함 (NOT NULL 제약 충족)

### 일관 실패 — 1개 ❌

**`e2e/mock/auth.spec.ts:121`**  
`S3T2 — /unauthorized 페이지가 렌더되고 홈 이동 버튼이 동작한다`

```
Error: getByRole('heading', { name: /권한 없음/ })
Expected: visible | element(s) not found (5000ms timeout)
```

**근본 원인 분석**:

```
page.goto(/#/unauthorized)  ← 풀 페이지 리로드
  └─ AuthContext 재초기화
       ├─ init(): getSession() → fetchIsAdmin() [요청 A]
       └─ onAuthStateChange(INITIAL_SESSION) → fetchIsAdmin() [요청 B]
            ↓ 두 요청이 동시 실행
  RequireAdmin: isAdmin === null → 스피너 표시 (UnauthorizedPage 미렌더)
  waitForLoadState('networkidle') → 요청 A 완료 시 idle 판정
  heading 체크(5000ms) → 요청 B 아직 진행 중 → isAdmin 여전히 null → 스피너
  timeout ❌
```

**시도한 접근과 결과**:
- JWT mock을 `exp=9999999999` 형식으로 교체 → token refresh 루프 제거 (부분 개선)
- `waitForLoadState('networkidle')` 추가 → networkidle이 양쪽 fetchIsAdmin 동시 완료 보장 못함
- 두 방법 조합 → 여전히 실패

**권장 해결책 (다음 세션)**:

옵션 A (근본 해결 — AuthContext 수정):
```typescript
// onAuthStateChange에서 INITIAL_SESSION 이벤트 무시
// (init()과 중복 실행 방지)
supabase.auth.onAuthStateChange(async (event, newSession) => {
  if (event === 'INITIAL_SESSION') return  // ← 추가
  ...
})
```

옵션 B (테스트 수정 — 빠른 해결):
```typescript
// heading timeout을 충분히 늘려 두 번의 admin 체크 완료 대기
await expect(page.getByRole('heading', { name: /권한 없음/ }))
  .toBeVisible({ timeout: 15000 })
```

옵션 C (테스트 구조 변경):
```typescript

```

  *_select_admin → EXISTS (FROM admin_users) ✅ (admin_users SELECT가 수정됐으므로 정상)
