# 🚀 LeeDeli 프로젝트 다음 할 일 (Next Steps)

이 문서는 프로젝트 구현 완료 후, 로컬 환경에서 실제 서비스를 구동하고 검증하기 위해 필요한 단계들을 정리한 가이드입니다.

---

## 1. Playwright를 이용한 브라우저 작동 확인

Playwright는 실제 브라우저 환경에서 사용자 시나리오를 테스트합니다. 프로젝트에 설정된 `mock` 테스트와 `smoke` 테스트를 통해 UI와 로직이 정상인지 확인하세요.

### ✅ 테스트 실행 순서
1. **Playwright 브라우저 설치 (최초 1회)**
   ```powershell
   npx playwright install
   ```

2. **Mock 테스트 실행 (가짜 데이터로 UI/로직 확인)**
   * Supabase 연동 없이도 UI가 정상인지 확인할 수 있습니다.
   ```powershell
   npm run test:mock
   ```

3. **Smoke 테스트 실행 (실제 연동 테스트)**
   * 아래 [2. Supabase 연동 설정]을 완료한 후 실행하세요.
   ```powershell
   npm run test:smoke
   ```

---

## 2. Supabase 연동 설정 (필수)

현재 모든 기능은 구현되었으나, 실제 데이터베이스와 연결되지 않은 상태입니다.

### ① 환경 변수 설정 (`.env`)
프로젝트 루트 폴더에 `.env` 파일을 생성하고, [Supabase Dashboard](https://supabase.com/dashboard)의 **Settings > API**에서 확인한 값을 입력하세요.

```env
VITE_SUPABASE_URL=당신의_Supabase_URL
VITE_SUPABASE_ANON_KEY=당신의_Supabase_Anon_Key
```

### ② 데이터베이스 테이블 생성 (Migration)
Supabase의 **SQL Editor**에 접속하여 아래 경로의 파일 내용을 순서대로 복사해 실행하세요.
1. `supabase/migrations/001_admin_users.sql` (관리자 테이블)
2. `supabase/migrations/002_core_tables.sql` (핵심 데이터 테이블)

---

## 3. 관리자 권한 부여 (Admin Authorization)

이 앱은 관리자만 특정 페이지(스케줄 생성 등)에 접근할 수 있습니다.

1. 앱 실행(`npm run dev`) 후 브라우저에서 회원가입/로그인을 진행합니다.
2. Supabase SQL Editor에서 본인의 계정 이메일을 `admin_users` 테이블에 추가하여 권한을 부여합니다.
   ```sql
   INSERT INTO admin_users (email) VALUES ('당신의_이메일@example.com');
   ```

## 4. 로컬 개발 서버 실행

모든 설정이 완료되었다면 아래 명령어로 앱을 실행하고 브라우저(`http://localhost:5173`)에서 확인하세요.

```powershell
npm run dev
```
