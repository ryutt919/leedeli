너는 세계 최고의 웹디자이너야. 컨텍스트의 폴더에 근무 스케줄 생성/관리 웹페이지를 만들어줘. 다음 요구사항을 충족하는 반응형 UI와 클라이언트-사이드 검증, 기본적인 데이터 저장(엑셀 또는 JSON export)을 포함해줘.

요구사항

1. 메인 페이지

   - 모던한 디자인(심플한 네비게이션, 카드 레이아웃, 반응형)
   - 버튼 2개(각 버튼 클릭 시 페이지 이동)
     - 버튼 A: "근무 스케줄 생성" 페이지로 이동
     - 버튼 B: "스케줄 관리/조회" 페이지로 이동(간단한 목록/필터 제공)

2. 근무 스케줄 생성 페이지

   - 연도(year) 및 월(month) 선택(드롭다운 또는 날짜픽커)
   - 근무 인원 관리 UI: 인원 수 선택 → 이름 입력 가능한 행들이 동적으로 생성
     - 각 인원에 대해:
       - 이름(필수)
       - 오픈/마감 태그(오픈, 마감 둘 다 체크 가능)
       - 오픈 필수/마감 필수 체크박스(중복 가능). 전체 규칙: 최소 한 명은 '오픈 필수', 최소 한 명은 '마감 필수'로 설정되어야 함
       - 특정 요구 휴무일 선택(달력에서 다중 날짜 선택)
       - 주 n일 근무 설정(숫자, 기본값 5)
   - 근무 규칙/상수:
     - 오픈조 출근 시간: 07:00
     - 마감조 출근 시간: 11:00
     - 1일 총 근무 인원: 3명
     - 1인 근무 시간: 8시간 + 근무 중 1시간 휴게(자동 계산)
     - 오픈/마감은 2교대(오픈/마감으로만 구분)
   - 생성 버튼 누르면 선택한 연도/월과 인원 설정을 기반으로 기본 스케줄 초안 생성(가능하면 알고리즘: 각 날짜에 3명 배치, 주 n일 근무/휴무 요청 반영, 오픈/마감 필수 인원 보장)
   - 생성 결과는 달력/테이블로 보여주고, 인원별/날짜별 편집 가능
   - 유효성 검사:
     - 이름 필수 입력
     - 연도/월 선택 필수
     - 각 날짜에 3명 미만 배치 불가
     - 오픈조/마감조에 반드시 최소 1명씩 '오픈 필수'/'마감 필수'가 할당되도록 검사
     - 개인 주 근무일 수가 초과되지 않도록 검사
     - 조건 만족 안하면 생성 불가표시와 이유 표시

3. 스케줄 관리/조회 페이지

   - 저장된 스케줄 목록(연도/월별)
   - 불러오기, 수정, 삭제, 엑셀/JSON 내보내기
   - 필터(연도, 월, 인원 이름 검색)

4. UI/UX 및 기술사항
   - 반응형, 접근성 고려(키보드로 항목 추가/편집 가능)
   - 클라이언트 사이드에서 즉시 미리보기 제공
   - 기본 스타일(모던): 깔끔한 색상 팔레트, 카드, 모달
   - 모듈화 적용
